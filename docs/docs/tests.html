TestHybridMlKem

Assuming the Key-Exchange algorithms "MLKem" and "CompositeMLKem" as well as the encryption algorithm "AES-GCM" are working well, it should be sufficient to verify key-import/-export and Encapsulate-/Decapsulate-calls are forwarded to the correct key exchange implementation, the CipherData class and the "AES-GCM" parameter are properly assigned.
